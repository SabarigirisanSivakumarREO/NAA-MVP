Two-Store Pattern + Warm-Up Mode
INTERNAL STORE (findings table)
• All findings, all statuses, all scopes
• Rejected findings visible
• Grounding metadata visible
• LLM trace refs visible
• WHO READS: Consultant dashboard, admin, Learning Service
• ACCESS: app.access_mode = 'internal'
PUBLISHED STORE (published_findings view)
• Only approved/auto-published findings
• No rejected, no held, no grounding details
• Heuristic ID stripped (IP protection)
• WHO READS: Client dashboard, MCP server, PDF export
• ACCESS: app.access_mode = 'published_only'
• ENFORCED: RLS at database layer
WARM-UP MODE: New clients → ALL findings held for consultant review. Auto-publish disabled until: audits ≥ 3 AND rejection rate < 25%.
