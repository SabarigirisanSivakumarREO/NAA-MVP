Core Design Philosophy

Controlled Intelligence

Not an autonomous agent set loose with a browser. A deterministic state machine with LLM-assisted reasoning at specific, bounded decision points.

Evidence-First

Every finding links to evidence: DOM elements, bounding boxes, computed styles, screenshots. No evidence = no finding. Period.

Three-Layer Hallucination Filter

CoT generation (~50% caught) → Self-critique review (~30% more) → Deterministic evidence grounding (~95% of remaining). What survives is real.

Deterministic Scoring

Severity, confidence, business impact, effort — all computed from data, never from LLM opinion. The LLM writes text; the code assigns numbers.

Consultant is Source of Truth

Raw AI output never reaches a client. Two-store pattern: internal store (everything) vs published store (consultant-approved only). Warm-up mode for new clients.

Reproducibility

Every audit run pins: prompt versions, model versions, heuristic versions, temperature (forced to 0). A finding that changed between runs is explainable, not mysterious.

Speaking notes
"There are 4 loops in the system — none unbounded:

1. Browse ReAct loop — perceive→reason→act→verify repeats until the page is stable. Confidence decays multiplicatively each step. Exits at confidence ≥0.7 or max_steps.

2. Escalation loop — if the self-critique says 'I think content is hidden behind a tab I couldn't see,' the system loops ONCE back to state exploration, re-explores, re-perceives, and re-analyzes. Maximum 1 cycle per page. Prevents unbounded re-exploration.

3. Page loop — the audit orchestrator's page_router loops through all pages in the queue. After each page's analysis is complete, it routes to the next page or signals audit_complete.

4. Learning feedback loop — consultant approve/reject/edit decisions feed the Learning Service, which adjusts heuristic reliability per client. Future audits for this client produce better-calibrated findings. This loop closes across audit runs, not within a single run."

Dual-Agent Model: Browse + Analyze

Browser Agent

Navigates, interacts, gathers evidence. Bounded autonomy.

Tier A: Deterministic$0/step · 90% of actions
Tier B: Guided (ReAct)$0.10/step · when rules fail
Tier C: Vision$0.30/step · last resort, max 5

Every action has a verify strategy. Confidence decays multiplicatively. HITL escalation at <0.3.

Analysis Agent

Evaluates pages against heuristics. Never touches the browser.

deep_perceiveFull page scan + multi-state
evaluate (CoT)LLM + chain-of-thought, temp=0
self_critiqueSeparate LLM call, adversarial
evidence_ground10 deterministic rules
score4D deterministic scoring

Analysis NEVER calls Playwright. Browse NEVER applies heuristics. Clean boundary.

Speaking notes
"The browser agent is not an LLM with browser tools. It is a deterministic state machine with LLM-assisted reasoning. 90% of browser actions use no LLM at all. The analysis agent only sees the perception data the browser produced — it never touches the browser directly. This separation is enforced at the code level, not by convention."

State Exploration — Seeing Hidden Content

Modern web pages hide 30-50% of CRO-relevant content behind tabs, accordions, modals, and variant selectors. We find it.

Pass 1: Heuristic-Primed

Each heuristic declares what UI states it needs. The exploration engine clicks only what's needed. Deterministic, cheap, targeted.

Example: Cialdini "social proof" heuristic says "look behind the Reviews tab." System clicks it.

Pass 2: Bounded-Exhaustive

If Pass 1 didn't find enough, auto-escalate: apply a 12-rule disclosure library (tabs, accordions, details, modals, variant selectors, filters, form validation). Max 15 states per page.

LLM fallback: 1 call max if <3 states found on an interactive page.

Meaningful-State Detection

Not every click reveals something new. We measure: text change >15%, new interactive elements >3, above-fold content change >10%, or CTA set changed. Non-meaningful states are discarded.

MultiStatePerception

All meaningful states are merged into a single perception view for heuristic evaluation. Every element tracked to its source state. GR-009 validates state provenance in findings.

Speaking notes
"Without state exploration, an audit of a product page misses the reviews tab, the size guide modal, the shipping accordion. That's 30-50% of the CRO-relevant content invisible to the analysis. Our two-pass model is rules-first — no LLM needed for 90% of exploration. The LLM is only a fallback when rules can't figure out what to click."

Two-Store Pattern + Warm-Up Mode

Raw AI output never reaches a client.

Internal Store

All findings, all statuses
Rejected findings visible
Grounding metadata, LLM traces
WHO: Consultant dashboard, admin
ACCESS: app.access_mode = 'internal'

Published Store

Only approved/auto-published
Heuristic ID stripped (IP protection)
No grounding details, no traces
WHO: Client dashboard, MCP server
ACCESS: app.access_mode = 'published_only'

Warm-Up Mode

New clients start with auto-publish DISABLED. ALL findings go to consultant review. After 3 audits AND rejection rate <25%, auto-publish activates. Consultant can override at any time.

Why: if the first 20 findings include 3 hallucinations, consultants lose trust forever. Warm-up earns trust first.
