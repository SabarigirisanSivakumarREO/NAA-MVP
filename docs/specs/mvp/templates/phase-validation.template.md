## §6 Trust calibration — what to spot-check by hand

When a human reviewer has 20 minutes to gain confidence in AI-built code, where should they look? List 3-5 specific files-with-line-numbers where the most error-prone logic lives. For each, name the kind of mistake that would land there.

```
1. <file:line-range> — <function name>
   Risk: <what could go subtly wrong here>
   How to verify: <what to look for in the code>

2. <file:line-range> — <function name>
   Risk: ...
   How to verify: ...

Example for Phase 1:

1. ContextAssembler.ts:131-163 — fitToTokenBudget shrink ladder
   Risk: stage ordering inverted, or double-tokenize hit, or shrink mutates input
   How to verify: stages must run in exact order [depth, top30→20, drop visual,
                  accept]; each stage builds a NEW model object (look for `{ ...model, ... }`
                  spread rather than direct mutation); diagnostics.warnings only pushed once

2. SoftFilter.ts (any line with arithmetic)
   Risk: additive `score -= weight` instead of multiplicative — R4.4 violation
   How to verify: grep for `score [+\-]=` should return ZERO matches; the only
                  arithmetic on `score` should be `*=` or expression product

3. BrowserManager.ts:36 + 113-149 — R9 boundary
   Risk: Playwright type leaks upstream; BrowserPage wrappers expose more than
         what spec.md documents
   How to verify: only `BrowserManager.ts` imports `'playwright'`; BrowserPage
                  has exactly the methods listed in BrowserEngine.ts header; new
                  methods need impact.md update

4. AccessibilityExtractor.ts (parseAriaSnapshotYaml)
   Risk: indent-based YAML parser miscounts depth or fails on quoted role names
   How to verify: hand-trace 3-4 sample lines; cyclic input should be caught by
                  `checkAxTreeDepth` BEFORE z.parse, never after

5. ScreenshotExtractor.ts:107-115 — width-aware gate (Stage 2.5 I1 fix site)
   Risk: gate accidentally inverted (only resize when small) or `||` becomes `&&`
   How to verify: read the predicate; oversizeBytes + oversizeWidth must
                  trigger via OR not AND
```

**Trust calibration heuristic:** if all 3-5 spot-checks pass, treat the rest of the phase as TRUSTED. If any fail, escalate to a deeper Stage 2.5 re-review.
