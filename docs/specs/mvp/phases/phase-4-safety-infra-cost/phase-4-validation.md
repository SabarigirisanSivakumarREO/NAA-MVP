## §2 Data flow — canonical LLM call pipeline

ASCII pipeline diagram for `AnthropicAdapter.complete(req)` — the most complex orchestrating function in Phase 4 (6 control-flow paths × 5 outcomes × atomic log on every path).

Conventions:
- Nouns in `[brackets]` are typed values
- Verbs without brackets are function calls
- `--finally-->` shows always-runs cleanup
- `(async)` annotates an awaited call
- Pino-logged events shown as `>> event:foo.bar` to the right

```
                   LLM CALL PIPELINE (AnthropicAdapter.complete)
                   =============================================

  req: LLMCompleteRequest ---+
   {operation, prompt,       |
    model, audit_run_id,     |
    client_id, temperature,  |
    budget_remaining_usd}    |
                             v
              TemperatureGuard.check(req)                       >> r10.check
                   |                                                (operation,
       ok ---------+--------- throw TemperatureGuardError            temperature)
                   |          |
                   |          v
                   |     #tryWriteRow llm_call_log              >> log.write
                   |     {outcome: 'temperature_blocked',          (outcome='temp')
                   |      cost_usd: 0, model: req.model, ...}
                   |          |
                   |          v
                   |       throw to caller
                   v
              BudgetGate.estimate(req) --> [estimated_cost_usd]
                   |
                   v
              BudgetGate.check(req, budget_remaining_usd)       >> r14.budget
                   |                                              (estimated_cost,
       ok ---------+--------- throw BudgetExceededError           remaining)
                   |          |
                   |          v
                   |     #tryWriteRow llm_call_log              >> log.write
                   |     {outcome: 'budget_blocked', cost: 0,    (outcome='budget')
                   |      model: req.model, audit_run_id, ...}
                   |          |
                   |          v
                   |       throw to caller
                   v
              #invokeAnthropicAPI(req) (async; 3-retry exp-backoff)
                   |  retries on 5xx / timeout / 429
                   |
       success ----+---- 3-retry exhausted (LLMUnavailableError)
                   |          |
                   |          v
                   |     #tryWriteRow llm_call_log              >> log.write
                   |     {outcome: 'unavailable', cost: 0,       (outcome='unavail')
                   |      model: req.model, audit_run_id, ...,
                   |      retry_count: 3}
                   |          |
                   |          v
                   |       throw to caller (v1.2 will fallback here)
                   v
              [response: AnthropicMessage]
              + [usage: {input_tokens, output_tokens}]
                   |
                   v
              compute cost_usd = usage * model_rate
                   |
                   v
              #tryWriteRow llm_call_log                          >> log.write
              {outcome: 'ok', cost_usd, prompt_tokens,             (outcome='ok')
               completion_tokens, duration_ms, cache_hit,
               model, audit_run_id, client_id}
                   |
                   v
              return LLMCompleteResponse
              {text, model, usage, cost_usd}
```
