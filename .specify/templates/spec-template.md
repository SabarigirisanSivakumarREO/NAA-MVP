These constraints apply to EVERY feature. Spec Kit must propagate them to plan.md and tasks.md downstream.

- **Tech stack pinned (architecture.md §6.4):** TypeScript 5 + Node 22 + Turborepo + pnpm 9 + Playwright + LangGraph.js + Claude Sonnet 4 + Postgres 16 + Drizzle + Hono 4 + Next.js 15 + shadcn/ui + Tailwind + Clerk + Cloudflare R2 + Sharp + Pino + Resend + Vitest + Playwright Test. Do NOT propose alternatives.
- **File locations pinned (architecture.md §6.5).** Any new file outside the structure requires ASK FIRST per R11.3.
- **Adapter pattern (R9):** every external dependency goes through an adapter. Direct imports of `@anthropic-ai/sdk`, Playwright, `pg`, Drizzle outside `packages/agent-core/src/adapters/` are FORBIDDEN.
- **Temperature = 0** on `evaluate` / `self_critique` / `evaluate_interactive` (R10 + R13). Enforced at adapter boundary by `TemperatureGuard`.
- **No `console.log` in production code** (R10.6). Pino logger with correlation fields (audit_run_id, page_url, node_name, heuristic_id, trace_id) only.
- **No conversion-rate predictions** (R5.3 + GR-007). Banned phrases regex-checked at grounding.
- **No UPDATE/DELETE on append-only tables** (R7.4): `audit_log`, `rejected_findings`, `finding_edits`, `llm_call_log`, `audit_events`.
- **Heuristic content is IP-protected** (R6). Never appears in API responses, logs, dashboards, LangSmith traces, or PR descriptions. Only `heuristic_id` references.
- **Findings are HYPOTHESES, not VERDICTS** (R5.1). Every finding survives 3 filters (CoT, separate-persona self-critique, 12 grounding rules) before reaching a consultant.
